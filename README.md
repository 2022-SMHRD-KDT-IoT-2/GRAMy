### Back-End

- 스네이크 케이스 (snake_case)
- 모든 단어를 소문자로 표기하고, 단어를 언더바(\_) 로 연결하는 방법

### Front-End

- 카멜 케이스 (camelCase)
- 첫단어는 소문자로 표기하지만, 이어지는 단어의 첫글자는 대문자로 표기하는 방법

## GIT

### Commit
- 커밋 메시지 대충 작성 금지 (뭐 변경했는지 알아볼 수 있게 작성)

### PUSH
- PUSH할 때 항상 단톡방에 말하고 PUSH
- 다른 사람이 작성했던 코드 수정할 시 항상 단톡방에 말하고 수정
